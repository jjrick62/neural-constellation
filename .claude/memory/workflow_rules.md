# 本项目工作流规则

## 三阶段修改流程（最高优先级）

每次用户要求修改代码：
1. **Brainstorm** — 想有没有更优实现，和用户讨论确认
2. **翻译** — 把用户的非专业需求翻译为精确技术语言，更新 `设计思路Claude/` 下对应文档
3. **讲解** — 用比用户稍专业一级的语言解释实现方式

禁止跳过任何步骤直接改代码。

## 文档同步

所有代码修改须同步更新 `实现机理详解.md`（根目录）。
设计团队文档在 `设计思路Claude/` 下，给 Claude 看。
用户看 `实现机理详解.md`。
