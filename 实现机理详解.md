## 一、粒子系统（星空背景）

### 设计理念

你看到的 activetheory 粒子不是"自己会发光的灯泡"，而是**空间里漂浮的微小尘埃，被场景里的灯光照亮后反光**。就像你在暗房间里打开手电筒，照到空气中的灰尘——灰尘自己不亮，是手电筒的光打在它上面才亮。

我们的粒子同理：
- 粒子在 3D 空间里有**固定位置**，不随机乱窜
- 场景里有几个**点光源**（金色暖光），粒子被这些光照到才亮
- 离光源近的粒子亮、远的暗（光的平方反比衰减）
- 离摄像机近的粒子偏暖金、远的偏冷蓝（大气透视效果）

### 粒子密度分布

不是均匀撒豆子，而是用**数学函数控制疏密**：

```
核心思路：多个高斯聚类中心 + 伪随机噪声

1. 在空间里设几个"聚类中心点"（比如场景三技能星座附近粒子密集）
2. 每个粒子的位置 = 离它最近的聚类中心 + 一定范围内的随机偏移
3. 偏移量用高斯分布（大部分粒子靠近中心，少量飞得远）
4. 再叠加一个小的伪随机噪声（sin/cos 组合函数），让分布看起来"自然"而不是"机器均匀"
```

效果：某些区域粒子密集（像星云），某些区域稀疏（像深空），整体有疏有密。

### 粒子大小

```
粒子大小 = 基础大小 × 距离衰减系数 + 噪声扰动

- 基础大小：根据粒子离聚类中心的距离决定（中心大、边缘小）
- 距离衰减：离摄像机越远越小（透视缩小）
- 噪声扰动：每个粒子用 sin(pos.x * 7.3 + pos.y * 11.1) 之类的小值偏移，
  让相邻粒子大小略有差异，不会齐刷刷一样大
```

这就是你说的"按某种函数加上小比例伪随机噪声做粒子大小设置"。

### 粒子颜色

```
粒子颜色 = 光源颜色 × 光照强度 × 摄像机距离色调

三个因变量：
1. 光源颜色：场景里那几个金色点光源的颜色（#D4A843, #B8942E）
2. 光照强度：粒子离光源越近越亮（平方反比：intensity = 1 / distance²）
3. 摄像机距离：近处偏暖金，远处偏冷蓝（模拟大气散射）
```

**不是粒子自己发光**——粒子没有自发光（emissive），是外部光源照到它然后反射。我们不用 AdditiveBlending（加法混合会让粒子自己像灯泡），改用 NormalBlending（正常混合，粒子融入场景）。

### 粒子由亮到暗的过渡

粒子的亮度主要由两个因素决定：
- 离场景里金色点光源的距离（近亮远暗）
- 离摄像机的距离（近亮远暗，带色偏）

点光源的位置会跟着场景变化——比如场景二展厅有一个主光源在节点群中央，场景三有一个光源在星座中心。粒子在光源附近密集且亮，远离光源的地方稀疏且暗。

## 二、光效系统

### 节点光晕

不再用"一个圈套着球"的 Fresnel 球壳，改用 **Sprite 柔光贴图**：

```
原理：一个永远朝向摄像机的透明贴图，上面画着径向渐变
- 中心白/金色，向外渐变成透明
- 叠在节点背后，看起来像节点的柔光发散
- 跟真实摄影里的镜头光晕一个道理
```

### 场景灯光

```
场景里有 3 个灯：
1. 环境光（AmbientLight）：暗蓝紫色，给整个场景打底，避免纯黑看不见
2. 主点光源（PointLight）：暖金色，放在当前场景的视觉焦点附近
3. 辅助点光源：琥珀色，放在侧面补光，让暗面不完全黑
```

灯的位置随场景切换而移动——场景一在文字背后，场景二在节点群中央，场景三在星座中心。

## 三、页面主体 3D 元素

### 节点（Node）

```
几何体：球体（SphereGeometry, 32段）
材质：MeshStandardMaterial（PBR 标准材质）
  - roughness: 0.2（光滑金属）
  - metalness: 0.9（强金属感）
  - color: 金色或琥珀色，看是不是核心节点

光晕：Sprite 柔光贴图，见上面"光效系统"
```

为什么不用 Icosahedron（二十面体）？面数太低，灯光打上去棱角分明，像塑料玩具。球体 32 段，灯光一打有圆润高光。

### 突触管道（Synapse Tube）

```
结构：三层叠加
1. 内芯管道：TubeGeometry，半径 1.5 倍，金色半透明 (opacity 0.35)
2. 光晕管道：TubeGeometry，半径 4 倍，金色极透明 (opacity 0.08)
3. 流动粒子：100 个金色小粒子沿管道曲线流动

三层叠一起的效果：一条发光的金色神经纤维，粒子在管道里流
```

### 项目卡片（Project Card）

```
用 Canvas 2D 画好文字 → 贴到 Three.js PlaneGeometry 上当纹理
- 暗色半透明底板 + 金色边框 + 项目名 + 描述 + 技术标签
- 始终朝向摄像机（或朝一个固定方向微倾）
```

### 文字（Scene1 和 Scene4）

```
同样用 Canvas 2D 画文字 → 贴到 PlaneGeometry
- VINCENT 用金色渐变填充
- 金句、副标题、联系方式都走这个路子
```

## 四、摄像机运动轨迹

### 设计理念

```
滚动条位置 (0% ~ 100%) → 映射到摄像机位置和朝向

不是"页面往下滚"，而是"摄像机在 3D 空间里上下飞行"
- 滚到 0%：摄像机在底部，抬头看 VINCENT
- 滚到 25%：摄像机上升，平视看项目神经网络
- 滚到 50%：摄像机继续上升，低头看技能星座
- 滚到 75%：摄像机回落，远看关于页
- 滚到 100%：回到初始位置
```

### 实现

```
CameraController 类：
- 预设 8 个关键帧（每个关键帧 = 滚动进度 + 摄像机位置 + 看哪里 + FOV）
- 用户滚动时，在两个最近的关键帧之间做平滑插值（easeInOutQuad）
- 插值完直接设置 camera.position 和 camera.lookAt
```

## 七、滚动交互

```
页面高度设成 400vh（4 倍视口高度）→ 可以滚 3 屏

每帧做的事（rAF 循环）：
1. 读 window.scrollY → 算成 0-1 进度
2. 进度传给 CameraController → 移动摄像机
3. 根据进度决定显示哪个场景（0-0.25=场景一, 0.25-0.5=场景二...）
4. 更新当前场景的粒子、节点、连线
5. 更新星空背景（粒子缓慢旋转）
6. 更新鼠标状态
7. 渲染一帧
```

## 九、粒子系统重新设计（翟伟鑫方案）

### 核心原则

1. **粒子不发光**——粒子是空间里的微小反射体，被外部光源照亮后才可见
2. **固定位置**——粒子在 3D 空间有固定坐标，不随机抖动
3. **密度有层次**——用聚类函数控制疏密，不是均匀撒
4. **大小有噪声**——每个粒子大小由基值 + 小比例伪随机噪声组成
5. **颜色由光源和距离决定**——光源颜色 × 光照衰减 × 摄像机距离色偏

### 密度分布（四分区独立控制）

```
四个高度分区各有自己的粒子配额，互不干扰：

  Z0 扉页 [-10 ~ -4]：10,000 粒 — 稀疏干净
  Z1 项目 [ -4 ~  2]：34,000 粒 — 中等
  Z2 技能 [  2 ~  8]：43,000 粒 — 最密集
  Z3 关于 [  8 ~ 14]：33,000 粒 — 适中

每个区内的粒子用柱坐标 (r, θ, y) 生成：
  - Y 在区内均匀分布
  - θ 360°均匀环绕
  - r 用 √random 偏向壁面（外密内疏）
```

### 大小噪声

```
基础大小 = 0.04 + (r / 4.0) × 0.06  （靠壁越大）
再叠加 sin/cos 位置噪声 ±15%
```

### 时间闪烁

```
每个粒子根据世界坐标算一个独立相位
用 sin(时间 × 2.5 + 相位) 做亮度波动
取 6 次幂让闪烁"锐利"——粒子像星星眨眼，不是平滑渐变
亮度在 60%~100% 之间波动
```

### 光照计算（Shader 里做）

```
在 Fragment Shader 中：

对每个粒子（即每个像素），遍历场景的光源：
  vec3 totalLight = vec3(0.0);
  for (每个光源) {
    float dist = length(粒子世界坐标 - 光源位置);
    float attenuation = 1.0 / (1.0 + dist * dist * 0.01); // 平方反比
    totalLight += 光源颜色 * attenuation;
  }

最终粒子颜色 = totalLight × 基础色 × 摄像机距离衰减
  - 基础色：粒子固有的暖白色
  - 摄像机距离衰减：近处偏暖金，远处偏冷蓝

// 摄像机距离色偏
float camDist = length(粒子世界坐标 - 摄像机位置);
float warmth = exp(-camDist * 0.02);
vec3 nearColor = vec3(1.0, 0.9, 0.7);  // 暖金
vec3 farColor  = vec3(0.4, 0.5, 0.7);  // 冷蓝
vec3 color = mix(farColor, nearColor, warmth) * totalLight;
```

### 渲染设置

```
不使用 AdditiveBlending（加法混合会让粒子像灯泡发光）
使用 NormalBlending + 透明度
depthWrite: false（粒子不写深度，避免遮挡其他物体）
```

## 十、开发脚手架（调试用）

### 做什么的

开发阶段需要一些"透视眼"能力——看清楚 3D 空间里的方位、自由移动摄像机检查每个角落。这些功能叫"脚手架"，正式发布前会关掉。

### 调试坐标轴

```
原点在圆柱底面中心 (0, -10, 0)
六根白色粗线（±X / ±Y / ±Z）各长 24 单位
每 4 单位一个刻度 + 白色数字标签（...-8, -4, 0, 4, 8...）
原点有个大球，每根轴端点有个小球

以后咱俩交流直接报坐标："摄像机去 (0, 5, 3)"
```

原理：用 CylinderGeometry（细圆柱体）拼成轴线，比 Three.js 自带的 AxesHelper 粗。数字标签用 Canvas 画好文字贴到 Sprite 上。`depthTest: false` 保证粒子再多也挡不住轴线。

### 自由摄像机

右上角有个按钮，点一下从 **SCROLL**（正常滚动模式）切到 **FREECAM**（自由飞行模式）。

自由模式下页面滚动被拦截，底部显示快捷键面板：

```
W/S       前进/后退（视线方向）
A/D       左移/右移
Q/E       左右转头
R         仰头 / T  低头
Space     上升 / Shift  下降
鼠标滚轮  前进/后退（带惯性，越滚越快）
```

切换回滚动模式时会回到原来的滚动位置。

这个功能的好处：你不用从页面顶部滚到底部才能看 Z3 区——直接飞过去就行。想看圆柱外面也可以飞出去（自由模式没有边界限制）。
