# 06 — 滚动映射与输入处理

## 页面滚动设置

```css
html, body {
  overflow-x: hidden;
  overflow-y: auto;       /* 允许垂直滚动 */
}

/* JS 设置 */
document.body.style.height = '400vh';  /* 4 倍视口高度 */
```

## 滚动进度计算

```
每帧 rAF 中读取：
  scrollY = window.scrollY
  maxScroll = document.body.scrollHeight - window.innerHeight
  rawProgress = maxScroll > 0 ? scrollY / maxScroll : 0
  progress = clamp(rawProgress, 0, 1)
```

## 进度到摄像机 Y 的映射

```
分段线性映射，各区分配不同的滚动行程权重：

function progressToCameraY(p):
  // p ∈ [0, 1]
  if p < 0.20:
    // Z0 区：20% 行程覆盖 Y 从 -10 到 -4（6 单位）
    t = p / 0.20
    return lerp(-10, -4, easeInOutCubic(t))
  elif p < 0.45:
    // Z1 区：25% 行程覆盖 Y 从 -4 到 2（6 单位）
    t = (p - 0.20) / 0.25
    return lerp(-4, 2, easeInOutCubic(t))
  elif p < 0.70:
    // Z2 区：25% 行程覆盖 Y 从 2 到 8（6 单位）
    t = (p - 0.45) / 0.25
    return lerp(2, 8, easeInOutCubic(t))
  else:
    // Z3 区：30% 行程覆盖 Y 从 8 到 14（6 单位）
    t = (p - 0.70) / 0.30
    return lerp(8, 14, easeInOutCubic(t))
```

## 缓动函数

```
easeInOutCubic(t):
  return t < 0.5 ? 4*t*t*t : 1 - pow(-2*t + 2, 3) / 2

easeInOutQuad(t):
  return t < 0.5 ? 2*t*t : 1 - pow(-2*t + 2, 2) / 2
```

## GSAP ScrollTrigger 配置

```
保留 GSAP ScrollTrigger 作为滚动驱动源（但不用它的进度值）：
  - gsap.to(scrollProxy, { progress: 1, ease: 'none',
      scrollTrigger: { trigger: document.body, start: 'top top',
                       end: 'bottom bottom', scrub: 0.5 }
    })

  - 不使用 ScrollTrigger 的 progress 做摄像机计算
  - 仅用 ScrollTrigger 保证 GSAP 的平滑滚动响应
  - 实际的摄像机 Y 计算使用上述分段映射函数 + DOM scrollY

原因：DOM scrollY 在任何情况下都是准确值，
      ScrollTrigger.progress 在初始化时可能不准确
```

## 每帧渲染管线

```
function renderFrame():
  // 1. 读滚动位置
  scrollY = window.scrollY
  p = clamp(scrollY / maxScroll, 0, 1)
  camY = progressToCameraY(p)

  // 2. 更新摄像机
  zoneIndex = getZoneIndex(camY)  // 0/1/2/3
  zoneT = getZoneLocalT(camY, zoneIndex)  // 区内归一化进度
  updateCameraTrajectory(zoneIndex, zoneT)

  // 3. 更新粒子照明（光源位置跟随摄像机）
  updateLightPositions(camera.position)

  // 4. 检测是否进入过渡区间
  blendInfo = checkBlendZone(camY)

  // 5. 更新场景内容
  if blendInfo.active:
    updateSceneContent(blendInfo.fromZone, 1 - blendInfo.weight)
    updateSceneContent(blendInfo.toZone, blendInfo.weight)
    screenTransition.update(blendInfo.weight)
  else:
    updateSceneContent(zoneIndex, 1.0)

  // 6. 更新粒子
  starfield.update(delta, camera.position)

  // 7. 渲染
  renderer.render(scene, camera)
  if screenTransition.active:
    screenTransition.render()
```

## 输入设备兼容

```
桌面端：
  - 鼠标滚轮：原生 scroll 事件（body overflow-y: auto）
  - 触摸板：同上
  - 键盘方向键：同上

移动端（后续适配）：
  - 触摸滑动：依赖 -webkit-overflow-scrolling: touch
  - 如需自定义触摸处理：添加 passive touch 事件监听
    document.addEventListener('touchmove', handler, { passive: true })

自定义光标：
  - DOM div #cursor，position: fixed
  - mousemove 事件更新 translate
  - 速度感应缩放 '+' 字符
```

## 性能优化点

```
1. 粒子不更新时跳过 Shader uniform 上传
2. 非活跃区的内容 Group 设置 visible=false（跳过渲染遍历）
3. renderer.setPixelRatio(Math.min(devicePixelRatio, 2)) 限制像素密度
4. 粒子使用 NormalBlending，减少 GPU fill rate 压力
5. 过渡特效仅在过渡期间分配资源，完成后立即 dispose
6. Canvas 纹理创建后不重复创建（缓存）
```
