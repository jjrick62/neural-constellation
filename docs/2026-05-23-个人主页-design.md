# Vincent 个人主页 — "神经星座" 设计方案

## 设计概念

**"神经星座"（Neural Constellation）** — 夜空中的星图本身就是一张神经网络。

- 星点 = 神经元节点
- 星座连线 = 神经突触
- 粒子沿连线流动 = 信号/脉冲传递
- 节点大小亮度 = 激活强度

四个场景在一个统一的深空宇宙中，通过 3D 相机飞行串联叙事：**做了什么 → 会什么 → 是谁**。

## 全局视觉体系

### 色彩

| 层级 | 色值 | 用途 |
|------|------|------|
| 主焦点 | 暖金 `#D4A843` | 核心节点（Vincent）、当前高亮项目、脉冲信号 |
| 次要节点 | 琥珀 `#B8942E` | 技术标签、外环技能、连线 |
| 背景星空 | 冷蓝白微光 `#6B7B8D` 附近 | 百万稀疏背景粒子 |
| 连线流动粒子 | 金→白渐变 | 突触连线 + 信号粒子拖尾 |
| 底色 | 纯黑 `#000000` | Canvas 背景 |

### 粒子密度三层（对照 activetheory.net 实测值）

| 层 | 密度 | 行为 |
|----|------|------|
| 背景星场 | 稀疏，100万+ | 缓慢漂浮，冷蓝白微光，大面积留黑，不抢视觉焦点 |
| 网络连线粒子 | 中等，沿路径 | 金色小点沿连线匀速 + curl noise 扰动，带短拖尾 |
| 焦点粒子 | 高密，局部 | 当前卡片/中心节点周围旋转环绕 |

### 统一动效语言

- **呼吸**：节点周期性明暗脉动（1.5-3 秒周期，随机相位打散，避免同步）
- **流动**：粒子沿连线方向匀速 + curl noise 偏移（不僵硬）
- **聚散**：摄像机靠近时粒子加速、远离时减速（空间深度感）
- **鼠标响应**：光标附近节点增强发光、星空粒子微排开

## 四场景航线

纵向滚动驱动 3D 相机在场景间飞行，GSAP ScrollTrigger 映射滚动进度到相机时间轴。

### 场景一：扉页

| 要素 | 描述 |
|------|------|
| 背景 | 纯黑底 + 100万+ 稀疏冷蓝白星场粒子缓慢漂浮 |
| 主视觉 | "VINCENT" 大字浮现在空间中央，金色渐变 `#D4A843 → #8B6914` |
| 副标题 | "大数据管理与应用 · 创造数据与代码的静谧建筑"，琥珀色小字 |

| 过渡 | 滚动触发粒子加速、RGB 色散增强，场景溶解 |

### 场景二：神经网络项目展厅

| 要素 | 描述 |
|------|------|

| 连线 | 核之间全互连，金色突触连线，脉冲信号沿连线跳跃传递 |
| 技术栈节点 | 每个项目核周围环绕该项目的技术小节点（3-5个），半径较小 |
| 卡片设计 | 半透明暗色底板 + 金色边框 + 项目名 + 简短描述 + 技术标签 |
| 当前项 | 居中的卡片放大、边框辉光增强、脉冲频率加快；其余缩小、亮度降低（景深） |
| 交互 | 卡片朝鼠标微倾（3D 视差），悬停金色光晕脉动，"VIEW" 按钮链接 GitHub |
| 鼠标响应 | 光标划过时空星粒子微排开（涟漪效果），最近节点亮度增强 |

### 场景三：技能星座

| 要素 | 描述 |
|------|------|
| 结构 | 中心辐射——Vincent 中心节点金色强光 |
| 内环（核心4） | 大节点 + 粗连线 + 强光：AI Agent 编排、Three.js 3D 可视化、WebSocket 实时通信、计算机视觉 |
| 外环（辅助8） | 小节点 + 细连线 + 微光：React、Python Flask、PWA、Flutter、Prompt Engineering、Canvas、MediaPipe、Git |
| 动效 | 整组星座缓慢自转（~0.1 rad/s），粒子沿连线双向 curl-noise 流动 |
| 层级感 | 内环节点大而亮（激活强度高），外环小（激活强度低），粒子流从内向外扩散 |
| 摄像机 | 从外环推进，穿过粒子层，最终定格在中心 Vincent 节点前方 |
| 鼠标响应 | 光标附近节点呼吸频率加快、连线粒子流速加大 |

| 层 | 技术 | 加载方式 | 用途 |

### 神经突触实现方案（零美术素材）

**混合方案：管道壳 + 内部粒子流**

| 元素 | 技术 | 说明 |
|------|------|------|
| 突触管道 | `THREE.TubeGeometry` + 贝塞尔曲线 | 暗金色半透明外壳（opacity 0.1-0.2），暗示连接存在但不抢眼 |
| 内部粒子流 | BufferGeometry Points + Curl Noise | 金色粒子沿管道内壁流动，curl noise 驱动不规则扰动 |
| 脉冲信号 | 沿路径移动的高亮小球 + 管道瞬间增亮 | 信号到达时粒子加速、管道辉光增强 |
| 管道粗细 | 靠近节点粗、中间细 | 贝塞尔路径上 scale 衰减，模拟信号衰减感 |

**节点本身：**

| 元素 | 技术 |
|------|------|
| 几何体 | `IcosahedronGeometry(1, 2)` — 低面数二十面体，比圆球有科技感 |
| 材质 | MatCap（单张贴图模拟复杂光照，1 DrawCall，activetheory 同款方案）|
| 辉光 | 外层稍大透明球 + 径向 gradient opacity，模拟光晕 |
| MatCap 贴图 | 程序生成 256×256 径向渐变 PNG |

**全站美术素材需求 = 零。** 所有视觉元素均由代码生成的几何体 + Shader 实现。

### 复用自 activetheory.net 的 Shader 技术

| 技术 | 来源 Shader | 搬运方式 |
|------|-------------|----------|
| RGB 色散后处理 | `rgbshift.fs` | 直接抄 — R/G/B 三通道偏移合成 |
| Curl Noise | `simplenoise.glsl` 中的 `curlNoise()` | 直接抄 — Bridson 2007 算法，640行 |
| HSV 色相偏移 | `rgb2hsv.fs` | 直接抄 — `rgb2hsv()` + `hsv2rgb()` |
| 鼠标涟漪 | `GlobalComposite.fs` 流体采样逻辑 | 简化 — sin/cos 距离衰减替代完整流体模拟 |
| Blend Modes | `blendmodes.glsl` | 直接抄 — ColorDodge、Glow、Screen |
| FXAA 抗锯齿 | `FXAA.fs` | 直接抄，用于最终输出 |

### 性能策略（对照 activetheory.net 生产标准）

| 策略 | 说明 |
|------|------|
| 粒子计算进 Web Worker | 百万级粒子位置更新在 Worker 线程，主线程只做渲染 |
| requestIdleCallback | 非当前场景的粒子初始化、缓存操作延迟到空闲帧 |
| MatCap 替代 PBR | 技能节点和项目卡片用 MatCap（单张贴图 = 1 个 DrawCall） |
| 字体子集化 | 中文只导出实际使用的字符，woff2 格式 |
| 粒子 LOD | 远离摄像机的粒子降低更新频率/简化计算 |
| 缓存版本号 | 资源 URL 带 `?v=时间戳`，浏览器强缓存 |
| 分层渲染 | 背景层、主场景层、UI 叠加层分别渲染，降低重绘范围 |
